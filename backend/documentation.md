## 📂 1. Files API
Префикс: `/api/v1/files`
Назначение: загрузка файлов пользователем, сохранение в БД и файловой системе, подготовка документов для сравнения.

### 📌 POST `/api/v1/files/upload/single`
Загрузка одного файла.

Описание
Используется для загрузки одного документа (PDF, DOCX, TXT, MD и т.д.).
Файл сохраняется в storage/, а метаданные — в таблицу documents.

Для чего используется
загрузка исходных документов для последующего сравнения;

загрузка файлов для анализа, хранения или отчётности.

Параметры
Параметр	Тип	Где	Описание
file	UploadFile	form-data	Загружаемый файл
Валидация
MIME‑тип должен быть разрешён (pdf, doc, docx, txt, md).

Размер файла ≤ MAX_FILE_SIZE.

Ответ`
json
{  
  "id": 12,
  "filename": "contract_v1.docx",
  "mime": "application/vnd.openxmlformats-officedocument.wordprocessingml.document",
  "size": 18233,
  "storage_path": "storage/abc123.docx"
}`
## 📂 2. Comparisons API
Префикс: `/api/v1/comparisons  `
Назначение: создание задач сравнения документов, получение статуса, просмотр изменений.

### 📌 POST `/api/v1/comparisons`
Создание нового сравнения.

Описание
Создаёт задачу сравнения между двумя или более документами.
Сохраняет запись в таблицу comparisons, привязывает документы через comparison_files, создаёт запись в reports.

Для чего используется
запуск анализа различий между версиями документов;

подготовка отчёта о различиях.

Тело запроса`
json
{
  "title": "Сравнение договоров",
  "file_ids": [12, 13],
  "options": {}
}`
Ответ`
json
{
  "id": 1,
  "title": "Сравнение договоров",
  "status": "queued",
  "summary": null,
  "risk_counts": null,
  "report_id": 5
}`
### 📌 GET `/api/v1/comparisons`
Получение списка всех сравнений (новые сверху).

Описание
Возвращает список всех сравнений, отсортированных по дате создания.

Для чего используется
отображение истории сравнений;

выбор сравнения для просмотра.

Ответ
json`
[
  {
    "id": 1,
    "title": "Сравнение договоров",
    "status": "finished",
    "created_at": "2026-03-19T12:39:49",
    "report_id": 5
  }
]`
### 📌 GET `/api/v1/comparisons/{comparison_id}`
Получение статуса и сводки сравнения.

Описание
Возвращает:

статус (`queued`, `running`, `finished`, `failed`);

сводку изменений;

количество изменений по уровням риска;

`report_id`.

Для чего используется
отображение результата сравнения;

переход к скачиванию отчёта.

### 📌 GET `/api/v1/comparisons/track`
Универсальный эндпоинт для получения изменений.

Описание
Работает в двух режимах:

Параметр	Режим	Что делает
`comparison_id` указан	режим сравнения	Возвращает изменения только для одного сравнения
`comparison_id` отсутствует	глобальный режим	Возвращает все изменения по всем сравнениям
Поддерживает поиск (q) и пагинацию.

Для чего используется
построение таблицы изменений;

глобальный поиск по всем изменениям;

аналитика.

Пример ответа`
json
{
  "items": [
    {
      "id": 33,
      "kind": "modified",
      "before": "Сторона обязуется...",
      "after": "Сторона имеет право...",
      "risk_level": "yellow",
      "created_at": "2026-03-19T12:40:00"
    }
  ],
  "page": 1,
  "total_pages": 3
}`

## 📂 3. Reports API
Префикс: `/api/v1/reports`  
Назначение: генерация и скачивание отчётов по сравнениям.

### 📌 POST `/api/v1/reports/comparisons/{comparison_id}/export`
Запуск генерации отчёта.

Описание
Помечает отчёт как generating.
В реальной системе — запускает фоновую задачу генерации PDF/DOCX.

Для чего используется
подготовка отчёта для скачивания.

Ответ`
json
{
  "report_id": 5,
  "status": "generating"
}`
### 📌 GET `/api/v1/reports/{report_id}/download`
Скачивание готового отчёта.

Описание
Возвращает файл отчёта, если:

`status == "ready"`

`storage_path` существует

Для чего используется
скачивание итогового документа сравнения.

## 📂 4. Search API
Префикс: `/api/v1  `
Назначение: поиск по документам и изменениям.

### 📌 GET `/api/v1/documents/search`
Поиск по документам и связанным изменениям.

Описание
Ищет:

по имени файла;

по текстам изменений (`before`, `after`);

по связанным актам (если есть интеграция с НПА).

Для чего используется
быстрый поиск нужного документа;

поиск по содержанию изменений.

### 📌 GET `/api/v1/change_items/search`
Поиск по изменениям.

Описание
Ищет по:

`before`

`after`

`risk_level`

Для чего используется
фильтрация изменений;
аналитика.
